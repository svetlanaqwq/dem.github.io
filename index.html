Структура доменного имени

Из чего состоит доменное имя?

Рассмотрим полный URL моего сайта:

https://svetlanaqwq.github.io/dem.github.io/

Важно понимать: только часть до первого слэша (/) — это доменное имя. Всё остальное — путь к ресурсу.

Таким образом, настоящее доменное имя (FQDN — Fully Qualified Domain Name):

svetlanaqwq.github.io

Разберём его по уровням иерархии DNS (читаем справа налево):

1. Домен верхнего уровня (TLD — Top-Level Domain):
.io — изначально национальный домен Британской территории в Индийском океане, но сейчас популярный в IT-сфере.
2. Домен второго уровня (SLD — Second-Level Domain):
github — принадлежит компании GitHub и используется для сервиса GitHub Pages.
3. Поддомен третьего уровня:
svetlanaqwq — это имя вашего аккаунта на GitHub. Оно делает ваш сайт уникальным в рамках github.io.
Важно! Часть /dem.github.io/ — это не домен, а путь (каталог или файл) на сервере. Она не участвует в DNS-иерархии и не влияет на доменное имя.

📊 Сравнение моделей

Модель Принцип работы Плюсы Минусы
Линейная регрессия Ищет прямую линию y = kx + b Быстрая, простая Не ловит сложные зависимости
Дерево решений Задает вопросы "Если-То" Легко интерпретировать Склонно к переобучению
Нейронная сеть Многослойная обработка с весами Ловит сложные паттерны Требует больше данных

🧠 Как работает нейронная сеть

🔹 Шаг 1: Входные данные

Входные признаки: ┌─────────────────────┐ │ gender │ → вес w1 │ lunch │ → вес w2 │ test preparation │ → вес w3 │ reading score │ → вес w4 └─────────────────────┘

🔹 Шаг 2: Функция активации (Сигмоида)

def sigmoid(x): return 1 / (1 + np.exp(-x)) # Преобразует любое число в диапазон от 0 до 1 # Добавляет нелинейность для обучения сложных зависимостей

🔹 Шаг 3: Архитектура сети

┌─────────┐ ┌──────────────┐ ┌─────────┐ │ ВХОД │ ──→ │ СКРЫТЫЙ СЛОЙ │ ──→ │ ВЫХОД │ │ (4 нейр.)│ │ (2 нейр.) │ │ (1 нейр.)│ └─────────┘ └──────────────┘ └─────────┘

🔹 Шаг 4: Обучение (Backpropagation)

Процесс обучения:
  1. Прямой проход → получаем прогноз
  2. Сравниваем с правильным ответом
  3. Считаем ошибку (MSE)
  4. Обратный проход → корректируем веса
  5. Повторяем много раз (эпохи)

🏗️ Архитектура нашей модели

# Параметры модели epochs = 1000 learning_rate = 0.1 input_dim = 4 # gender, lunch, test prep, reading score hidden_dim = 2 # 2 нейрона в скрытом слое output_dim = 1 # 1 нейрон на выходе (math score)

-- Создание таблицы Врачи
CREATE TABLE doctors (
    doctor_id INT PRIMARY KEY AUTO_INCREMENT,
    full_name VARCHAR(100) NOT NULL,
    age INT,
    experience_years INT,
    patients_count INT DEFAULT 0,
    education TEXT,
    city VARCHAR(50),
    birthplace VARCHAR(100),
    operations_count INT DEFAULT 0
);

CREATE TABLE nurses (
    nurse_id INT PRIMARY KEY AUTO_INCREMENT,
    full_name VARCHAR(100) NOT NULL,
    age INT,
    education TEXT,
    experience_years INT,
    position ENUM('старшая', 'младшая'),
    city VARCHAR(50),
    birthplace VARCHAR(100),
    wards_count INT DEFAULT 0

-- Создание таблицы Операции
CREATE TABLE operations (
    operation_id INT PRIMARY KEY AUTO_INCREMENT,
    operation_name VARCHAR(100) NOT NULL,
    operation_date DATE,
    patient_name VARCHAR(100),
    shift_number INT,
    operation_time TIME,
    patient_data TEXT,
    operating_room_number INT,
    diagnosis TEXT
);

💡 Примеры полезных запросов

-- 1. Показать все данные по сущности Врач
SELECT * FROM doctors;

-- 2. Найти врачей с опытом более 10 лет
SELECT full_name, experience_years, operations_count 
FROM doctors 
WHERE experience_years > 10
ORDER BY operations_count DESC;

-- 3. Показать всех медсестер должности "старшая"
SELECT full_name, wards_count 
FROM nurses 
WHERE position = 'старшая';

-- 4. Статистика операций по врачам
SELECT 
    d.full_name AS врач,
    COUNT(o.operation_id) AS количество_операций
FROM doctors d
LEFT JOIN operations o ON d.doctor_id = o.doctor_id
GROUP BY d.doctor_id;

-- 5. Найти медсестер, контролирующих более 30 палат
SELECT full_name, wards_count 
FROM nurses 
WHERE wards_count > 30;
